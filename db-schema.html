NPU Benchmark 平台 — 数据库表结构

平台简介

NPU Benchmark 平台用统一口径记录不同 NPU 硬件上的推理性能数据，支持跨模型、跨硬件、跨精度的横向对比与历史复用。核心业务对象有四个：

对象 含义
Solution（推理方案） 一份「模型 × 硬件 × 精度 × 配置」的可复现组合
Run（测试任务） 一次具体的 benchmark 执行，记录触发方式与执行状态
Result（测试结果） 一次 Run 产生的性能指标（延迟、吞吐、内存、功耗等）
Comparison（对比） 在 2–8 个 Solution 之间做横向对比，可生成分享链接

表总览

当前共 5 张已落地的表：inference_solutions、artifacts、benchmark_runs、benchmark_results、comparisons。

model_id / device_id / created_by 当前为裸 UUID 引用，对应的模型注册表、设备注册表、用户表将在后续版本补齐，届时会加 FK 约束。

1. inference_solutions — 推理方案

列 类型 可空 默认 说明
id UUID NOT NULL uuid4() 主键
name VARCHAR(200) NOT NULL — 方案名称
description TEXT NOT NULL "" 描述
status ENUM solution_status NOT NULL 'draft' draft / published / archived
version VARCHAR(20) NOT NULL '1.0.0' 版本号
model_id UUID NOT NULL — 引用模型注册表
device_id UUID NOT NULL — 引用设备注册表
created_by UUID NOT NULL — 创建者（引用用户表）
conversion JSONB NOT NULL {} 转换/编译参数（含精度 precision、量化参数、编译参数等）
runtime JSONB NOT NULL {} 运行时参数
tags TEXT[] NOT NULL {} 标签数组
created_at TIMESTAMPTZ NOT NULL now() 创建时间
updated_at TIMESTAMPTZ NOT NULL now() 更新时间（自动维护）

关系：

  • 1 → N → artifacts（级联删除）
  • 1 → N → benchmark_runs（级联删除）

2. artifacts — 方案产物

存放与方案关联的模型二进制、配置、脚本、日志、报告等文件元信息。

列 类型 可空 默认 说明
id UUID NOT NULL uuid4() 主键
solution_id UUID NOT NULL — FK → inference_solutions.id ON DELETE CASCADE
type ENUM artifact_type NOT NULL — model_binary / config / script / log / report
filename VARCHAR(500) NOT NULL — 文件名
storage_path TEXT NOT NULL — 存储路径（本地路径或对象存储 URI）
size_bytes INTEGER NULL — 文件大小
checksum VARCHAR(64) NULL — 文件校验和（如 sha256）
uploaded_at TIMESTAMPTZ NOT NULL now() 上传时间

3. benchmark_runs — 测试任务

列 类型 可空 默认 说明
id UUID NOT NULL uuid4() 主键
solution_id UUID NOT NULL — FK → inference_solutions.id
device_id UUID NOT NULL — 实际运行设备
status ENUM run_status NOT NULL 'pending' pending / running / completed / failed
trigger ENUM run_trigger NOT NULL 'manual' manual / auto / scheduled
test_config JSONB NOT NULL {} 运行时测试参数（场景、并发数等）
environment JSONB NOT NULL {} 环境信息（SDK 版本、驱动版本等）
error_message TEXT NULL — 失败原因
started_at TIMESTAMPTZ NULL — 开始执行时间
finished_at TIMESTAMPTZ NULL — 完成时间
created_by UUID NULL — 触发者
created_at TIMESTAMPTZ NOT NULL now() 创建时间

关系：

  • N → 1 → inference_solutions
  • 1 → 0..1 → benchmark_results（级联删除）

4. benchmark_results — 测试结果

每个 Run 最多对应一条 Result。

列 类型 可空 默认 说明
id UUID NOT NULL uuid4() 主键
run_id UUID NOT NULL — FK → benchmark_runs.id ON DELETE CASCADE
latency_p50_ms NUMERIC(10,3) NULL — P50 延迟（毫秒）
latency_p95_ms NUMERIC(10,3) NULL — P95 延迟（毫秒）
latency_p99_ms NUMERIC(10,3) NULL — P99 延迟（毫秒）
throughput NUMERIC(12,3) NULL — QPS
memory_peak_mb NUMERIC(10,2) NULL — 内存峰值（MB）
power_mw NUMERIC(10,2) NULL — 功耗（毫瓦）
ttft_ms NUMERIC(10,3) NULL — 首 token 延迟（LLM 场景）
tps NUMERIC(10,3) NULL — Token 吞吐量（LLM 场景）
accuracy JSONB NOT NULL {} 精度指标（key 由任务类型决定，如 top1 / top5 / mAP50）
raw_log_path TEXT NULL — 原始日志路径
created_at TIMESTAMPTZ NOT NULL now() 创建时间

5. comparisons — 对比

列 类型 可空 默认 说明
id UUID NOT NULL uuid4() 主键
name VARCHAR(200) NOT NULL — 对比名称
description TEXT NOT NULL "" 描述
solution_ids UUID[] NOT NULL — 参与对比的 Solution（业务约束 2–8 个）
baseline_id UUID NOT NULL — 基准 Solution（须为 solution_ids 之一）
metrics_selected TEXT[] NOT NULL {} 选择展示的指标名
run_snapshot JSONB NOT NULL {} {solution_id: run_id} 快照，保证对比可复现
notes TEXT NOT NULL "" 备注
created_by UUID NOT NULL — 创建者
created_at TIMESTAMPTZ NOT NULL now() 创建时间
updated_at TIMESTAMPTZ NOT NULL now() 更新时间（自动维护）
shared BOOLEAN NOT NULL false 是否公开分享
share_token VARCHAR(64) NULL — UNIQUE，分享链接 token
share_expires_at TIMESTAMPTZ NULL — 分享过期时间

solution_ids 与 run_snapshot 通过 UUID / JSONB 软引用其它表，不设外键，便于历史快照独立保留。

6. 枚举类型

枚举名 取值 使用位置
solution_status draft / published / archived inference_solutions.status
artifact_type model_binary / config / script / log / report artifacts.type
run_status pending / running / completed / failed benchmark_runs.status
run_trigger manual / auto / scheduled benchmark_runs.trigger

7. 关系图

inference_solutions ──┬── 1:N ──> artifacts            (ON DELETE CASCADE)
                      └── 1:N ──> benchmark_runs       (ON DELETE CASCADE)
                                       │
                                       └── 1:0..1 ──> benchmark_results  (ON DELETE CASCADE)

comparisons  ─── 通过 solution_ids[] / run_snapshot JSONB 软引用 solutions 和 runs

8. 命名与约定

  • 主键：所有业务表均为 UUID，由应用层生成（uuid4()）。
  • 时间戳：统一 TIMESTAMPTZ，默认 now()；updated_at 由 ORM 在更新时自动写入。
  • JSONB 字段：用于存放结构灵活、查询要求不高的配置或扩展指标，便于演进而无需频繁迁移。
  • 软删除：Solution 通过 status = 'archived' 实现，物理删除仅在级联场景发生。
  • 级联删除：删除 Solution 会连带删除其 Artifacts、Runs 及 Runs 下的 Results。
